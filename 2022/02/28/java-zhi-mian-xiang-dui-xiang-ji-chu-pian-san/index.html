Java之面向对象基础篇三

Java
发布日期:   2022-02-28
文章字数:   2.2k

Java之面向对象基础篇三

一、多态

多态：同一个对象在不同时刻表现出来的不同形态

1、多态的前提和体现

  • 有继承/实现关系

  • 有方法重写

  • 有父类引用指向子类对象，

    Parent p = new Child();

2、成员访问特点

  • 成员变量：编译看左边，执行看左边，即编译和执行都找父类中的变量
  • 成员方法：编译看左边，执行看右边，即编译找父类中有无方法，执行时执行子类中的同名方法

3、好处和弊端

  • 好处：提高了程序的扩展性。定义方法的时候，使用父类型作为参数，将来在使用的时候，使用具体的子类型参与操作
  • 弊端：不能使用子类的特有功能

4、多态中的转型

  • 向上转型：从子到父，父类引用指向子类对象
  • 向下转型：从父到子，父类引用转为子类对象
// 向上转型
Animal a = new Cat();

// 向下转型
Cat c = (Cat)a;

转型内存图

二、抽象类

在java中，一个没有方法体的方法应该定义为抽象方法，而类中如果有抽象方法，该类必须定义为抽象类

// 抽象类
public abstract class Animal{
    
    // 抽象方法
	public abstract void eat();
    
    // 普通方法
    public void sleep(){
        System.out.println("睡觉");
    }
}

public class Cat{
    
    // 子类必须重写父类的抽象方法
    @Override
    public void eat(){
        System.out.println("吃");
    }
}

// 用多态的方式实例化抽象类
Animal a = new Cat();
a.eat();  // 输出 吃

1、抽象类特点

  • 抽象类不能被实例化(初学者很容易犯的错)，如果被实例化，就会报错，编译无法通过。只有抽象类的非抽象子类可以创建对象。可以用多态的方式实例化
  • 抽象类中不一定包含抽象方法，但是有抽象方法的类必定是抽象类
  • 抽象类中的抽象方法只是声明，不包含方法体，就是不给出方法的具体实现也就是方法的具体功能
  • 构造方法，类方法（用 static 修饰的方法）不能声明为抽象方法
  • 抽象类的子类必须给出抽象类中的抽象方法的具体实现，除非该子类也是抽象类
  • 抽象类和抽象方法必须使用abstract关键字修饰

2、抽象类成员特点

  • 成员变量：可以是变量，可以是被final修饰的常量
  • 构造方法：有构造方法，但不能实例化，构造方法用于子类访问父类数据的初始化
  • 成员方法：有抽象方法，限定子类必须完成某些动作；有非抽象方法，提高代码复用性

三、接口

接口就是一种公共的规范标准，只要符合规范标准，大家都可以通用

Java中的接口更多的体现在对行为的抽象

1、接口的特点

  • 接口用关键字interface修饰

    public interface 接口名(){}

// 例子
public interface Jumping(){
    public void jump();
}

  • 类实现接口用implements表示

    public class 类名 implements 接口名(){}

// 例子
public class Cat implements Jumping(){
    
    @Override
    public void jump(){
        System.out.println("跳")
    }
}

  • 接口不能实例化，但可以通过类对象实例化，叫接口多态

    Jumping j = new Cat();
j.jump();

  • 接口的实现类要么重写接口中所有的抽象方法，要么是抽象类

2、接口的成员特点

  • 接口中成员方法是隐式抽象的,接口中的方法会被隐式的指定为 public abstract（只能是 public abstract，其他修饰符都会报错）
  • 接口中可以含有变量，但是接口中的变量会被隐式的指定为 public static final 变量，即默认是常量（并且只能是 public，用 private 修饰会报编译错误）
  • 接口中的方法是不能在接口中实现的，只能由实现接口的类来实现接口中的方法
  • 接口没有构造方法

注：

  • JDK 1.8 以后，接口允许包含具体实现的方法，该方法称为”默认方法”，默认方法使用 default 关键字修饰

    // 默认方法，有方法体，不强制重写，但可以在实现类重写，不过要去掉 default
public default void 方法名(){
    // 方法体
}

// 静态方法，有方法体，只能通过接口名调用
public static void 方法名(){
    // 方法体
}

接口名.方法名();

  • JDK 1.9 以后，允许将方法定义为 private，使得某些复用的代码不会把方法暴露出去

    // 括号表示可以加，加了就为私有静态方法
private (static) 返回值类型 方法名(参数列表){
    // 方法体
}

3、类和接口的关系

类描述对象的属性和方法，接口则包含类要实现的方法。

  • 类和类的关系：继承关系，只能单继承，但可以多重继承
  • 类和接口的关系：实现关系，可以单实现，也可以多实现，还可以在继承一个类的同时实现多个接口
  • 接口和接口的关系：继承关系，可以单继承，也可以多继承

4、抽象类和接口的区别

  • 抽象类中的方法可以有方法体，就是能实现方法的具体功能，但是接口中的方法不行
  • 抽象类中的成员变量可以是各种类型的，而接口中的成员变量只能是 public static final 类型的
  • 一个类只能继承一个抽象类，而一个类却可以实现多个接口
  • 抽象类是对事物的抽象，接口是对行为的抽象

快捷键：idea生成构造方法的快捷键是Alt+Insert

四、内部类

内部类：在一个类中定义一个类

定义格式：

public class 类名{ 			// 外部类
	修饰符 class 类名{		  // 内部类
	}
}

按照内部类在类中定义的位置不同，分为如下两种形式：

  • 在类的成员位置：成员内部类
  • 在类的局部位置（方法中定义的类）：局部内部类

1、内部类的访问特点

  • 内部类可以直接访问外部类的成员，包括私有
  • 外部类要访问内部类成员，必须创建对象
public class Outer{
	
    private int num = 10;
    
    // 成员内部类
    public class Inner{
        
        public void show(){
            System.out.println(num);	// 直接访问
        }
    }
    
    public void method(){
        
        show();   // 报错，外部类无法访问内部类
        
        // 创建对象调用
        Inner i = new Inner();
        i.show();
    }
}

2、成员内部类

成员内部类，外界如何创建对象使用

格式：

外部类名.内部类名 对象名 = 外部类对象.内部类对象;

public static void main(String[] args){
    // 创建内部类对象，并调用方法
    Outer.Inner oi = new Outer().new Inner();
    oi.show();
}

还有另外一种形式（常用）：

public class Outer{
	
    private int num = 10;
    
    // 成员内部类，当将内部类设为 private
    private class Inner{
        
        public void show(){
            System.out.println(num);	// 直接访问
        }
    }
    
    // 用方法对内部类调用
    public void method(){
        Inner i = new Inner();
        i.show();
    }
}

public static void main(String[] args){
    // 通过创建外部类，调用方法实现
    Outer o = new Outer();
    o.method();
}

3、局部内部类

局部内部类是在方法中定义的类，外界无法使用，需要在方法内部创建对象并使用，可以直接访问外部类的成员，也可以访问方法内的局部变量

public class Outer{
	
    private int num = 10;
    
    public void method(){
        
        int num2 = 20;
        
        // 局部内部类
        class Inner{
            public void show(){
                System.out.println(num);
                System.out.println(num2);
            }
        }
        
        // 在方法中创建对象调用
        Inner i = new Inner();
        i.show();
    }
}

public static void main(String[] args){
    // 通过创建外部类，调用方法实现
    Outer o = new Outer();
    o.method();
}

4、匿名内部类

前提：存在一个类或者接口，类可以是具体类或者抽象类

格式：

new 类名或者接口名(){
	重写方法;
};

// 匿名内部类
new Inter(){
    @Override
	public void show(){}
};

// 调用类中 show 方法
new Inter(){
    @Override
	public void show(){}
}.show();

Inter i = new Inter(){
    @Override
	public void show(){}
};

i.show();

本质：是一个继承了该类或者实现了该接口的子类匿名对象，是局部内部类的特殊形式

标准形式：

// 接口
public interface Inter{
    void show();
}

// 外部类
public class Outer{
    
    public void method(){
        
        // 匿名内部类
        Inter i = new Inter(){
            @Override
            public void show(){}
        };
        i.show();
    }
}

// 测试方法
public static void main(String[] args){
    // 通过创建外部类，调用方法实现
    Outer o = new Outer();
    o.method();
}
